# k-sajja-agents

![k-sajja-agents cover](./assets/cover.png)

한국 전문직을 위한 오픈소스 AI 에이전트 스킬 레지스트리.

`k-sajja-agents`는 변리사, 변호사, 세무사, 의사, 공인중개사처럼 한국 전문직의 사고방식과 업무 흐름을 AI 에이전트가 따라갈 수 있도록 정리하는 오픈소스 스킬 저장소입니다.

## 한 줄로 말하면

전문직 종사자와 함께, 한국형 직업 에이전트가 사용할 수 있는 `Agent Skill`을 무료 오픈소스로 만들어가는 프로젝트입니다.

## 왜 필요한가

웬만한 정보 탐색, 초안 작성, 요약, 비교, 체크리스트 작업은 AI가 대신 처리하기 시작했습니다. 사용자는 더 이상 처음부터 전문가에게 질문하지 않고, 먼저 AI에게 묻고 스스로 문제를 정리합니다.

하지만 AI에도 명확한 한계가 있습니다.

- AI는 재판에서 변론할 수 없습니다.
- AI는 약을 처방할 수 없습니다.
- AI는 세무 신고, 법률 대리, 의료 판단, 중개 책임을 사람처럼 대신할 수 없습니다.
- AI는 실제 사건과 이해관계에 대한 최종 책임을 지지 않습니다.

그래서 이 프로젝트는 AI가 잘할 수 있는 영역은 스킬로 정리하고, AI가 넘어설 수 없는 지점은 명확히 구분하는 것을 중요하게 봅니다.

## Agent Skill이란?

`Agent Skill`은 AI가 특정 방식으로 생각하고 행동하도록 만드는 작업 단위입니다.

예를 들어 변호사 에이전트라면 계약서를 받을 때 바로 답을 쓰는 것이 아니라, 당사자, 계약 유형, 위험 조항, 누락 조항, 협상 포인트를 순서대로 확인하도록 만들 수 있습니다. 세무사 에이전트라면 비용 자료를 보고 증빙 유형, 공제 가능성, 신고 전 확인사항을 먼저 정리하도록 만들 수 있습니다.

이 저장소는 이런 직업별 사고 과정을 `SKILL.md` 형태로 축적합니다.

아래 전문직 에이전트 패키지의 기본 디렉토리 골격을 제공합니다. 실제 스킬은 각 직역별 실무 흐름에 맞춰 순차적으로 추가합니다.

| 직업 패키지 | 다룰 수 있는 업무 |

| `patent-attorney/` | 특허 명세서 검토, 선행기술 조사, 의견제출통지 대응 초안 |
| `lawyer/` | 계약서 검토, 조항 요약, 리스크 체크, 법령 기반 질의응답 |
| `tax-accountant/` | 비용 정리, 증빙 분류, 세무 체크리스트, 신고 자료 검토 |
| `doctor/` | 증상 정리, 진료 전 문진 준비, 검사 결과 질문 목록 작성 |
| `pharmacist/` | 복약 상담 준비, 의약품 정보 정리, 병용 주의사항 체크 |
| `certified-labor-attorney/` | 근로계약, 임금, 해고, 직장 내 괴롭힘 관련 체크리스트 |
| `real-estate-agent/` | 매물 비교, 전월세 조건 검토, 특약 체크, 실거래가 리서치 |
| `appraiser/` | 감정평가 자료 정리, 비교 사례 검토, 평가 질문 목록 작성 |
| `customs-broker/` | 수출입 서류 체크, HS 코드 검토, 통관 절차 안내 |

## 디렉토리 규칙

루트 디렉토리에는 직업 이름을 둡니다. 각 직업 디렉토리 안에는 기여자 이름 또는 업체명을 영문 슬러그로 만든 기여자 디렉토리를 둡니다. 각 기여자 디렉토리 안에 해당 기여자가 만든 스킬 디렉토리를 둡니다.

```text
lawyer/
├── .gitkeep
├── hong-gildong/
│   ├── PROFILE.md
│   └── lawyer-contract-review/
│       └── SKILL.md
└── startup-law-lab/
    └── lawyer-legal-summary/
        └── SKILL.md
```

기본 경로는 `직업명/기여자-slug/직업명-스킬이름/SKILL.md` 형식을 사용합니다. 기여자 슬러그는 작성자 이름, 사무소명, 병원명, 업체명 등을 소문자 영문, 숫자, 하이픈(`-`)으로 정규화합니다.

```text
lawyer/hong-gildong/lawyer-contract-review/SKILL.md
tax-accountant/kim-tax-office/tax-accountant-vat-check/SKILL.md
patent-attorney/ip-seoul/patent-attorney-prior-art-search/SKILL.md
real-estate-agent/gangnam-home/real-estate-agent-lease-check/SKILL.md
```

직업 디렉토리에는 기여자 디렉토리와 빈 디렉토리 유지를 위한 `.gitkeep`만 둡니다. 기여자 디렉토리에는 선택적으로 `PROFILE.md`를 둘 수 있습니다. 같은 기여자가 여러 스킬을 만들 때 반복되는 전문가 소개, 업체 정보, 연락처, 상담 안내를 공통으로 관리하기 위한 파일입니다. 개별 스킬에도 필요한 범위에서 전문가 프로필과 연락처를 다시 적을 수 있습니다.

스킬이 외부 자료, 공개 문서, 오픈소스 코드를 참고했다면 `SKILL.md` 안에 출처와 라이선스를 함께 기록합니다.

예외적으로 레포 운영과 기여를 돕는 메타 스킬은 루트에 둘 수 있습니다. 예를 들어 `sajja-skill-creator/`는 전문직 기여자가 문답을 통해 직업별 스킬 초안을 만들도록 돕는 메타 스킬입니다. 메타 스킬도 한국어 우선, MIT 라이선스, agentskills 스펙, 플랫폼 독립 원칙을 따릅니다.

## 기여 방식

1. 추가할 직업 패키지를 정합니다.
2. 직업 디렉토리 안에 본인 또는 업체의 기여자 슬러그 디렉토리를 만듭니다.
3. 기여자 디렉토리 안에 `직업명-스킬이름` 형식의 스킬 디렉토리를 만듭니다.
4. `SKILL.md`에는 작업 절차, 입력 예시, 출력 형식, 제한사항을 한국어로 작성합니다.
5. 같은 기여자의 공통 소개가 필요하면 기여자 디렉토리에 `PROFILE.md`를 추가합니다.
6. 전문지식이 필요한 경우 법령, 판례, 고시, 기관자료, 학회 가이드라인 등 공개 출처의 확인 경로를 적습니다.
7. AI가 할 수 있는 일과 실제 전문가가 해야 하는 일을 구분합니다.
8. 외부 자료를 참고했다면 `SKILL.md` 안에 출처와 라이선스를 남깁니다.
9. 레포에 작성하는 원문 기여가 MIT 라이선스로 공개된다는 점을 확인합니다.
10. PR 설명에는 기여자 또는 업체의 공개 프로필, 관리자 확인용 연락 경로, 자격 또는 경력 확인 방법을 적습니다. 공개 PR에는 주민등록번호, 자격증 사본, 민감한 자격번호, 의뢰인 정보, 환자 정보, 내부 문서를 올리지 않습니다.

## 전문가 프로필

스킬 기여자는 원하면 본인의 전문가 프로필을 기여자 디렉토리의 `PROFILE.md` 또는 개별 스킬 안에 포함할 수 있습니다.

프로필은 프로젝트의 필수 요소가 아니라 선택사항입니다. 목적은 사용자가 스킬의 작성 주체와 검토 배경을 이해하도록 돕는 데 있습니다. 필요한 경우, AI만으로 해결하기 어려운 문제에서 실제 전문가에게 문의할 수 있는 정보를 함께 제공할 수 있습니다.

예시:

```text
# lawyer/hong-gildong/PROFILE.md

## 전문가 연결

- name: 홍길동 변호사
- field: 스타트업 계약, 개인정보, 서비스 약관
- region: 서울
- website: https://example.com
- contact: hello@example.com
- note: 이 스킬은 일반 정보 제공용이며, 구체적인 사건 검토는 별도 상담이 필요합니다.
```

전문가 프로필을 포함하는 경우 각 직역의 광고·표시 규정을 존중해야 합니다. 사용자가 광고, 소개, 일반 정보 제공, 전문 상담의 경계를 이해할 수 있도록 명확하게 작성해야 합니다.

스킬이 AI의 한계에 도달했을 때 작성자 또는 소속 전문가/업체로 연결될 수 있도록, 필요한 경우 연락처와 상담 안내를 함께 둘 수 있습니다. 단, 사용자가 AI 답변을 개별 자문이나 진료, 대리, 신고, 중개 책임으로 오해하지 않도록 주의 문구를 포함해야 합니다.

PR 단계에서는 관리자가 작성자의 전문성 또는 소속을 확인할 수 있어야 합니다. 기여자는 공개 가능한 프로필, 홈페이지, 예약 페이지, 소속 정보, 관리자 확인용 이메일 등을 PR 설명에 적습니다. 민감한 신원확인 자료는 공개 저장소에 올리지 말고, 필요한 경우 관리자와 별도 경로로 확인합니다.

## 작성 가이드

- 공개 가능한 자료만 포함합니다.
- 이 저장소에 직접 작성하는 원문은 MIT 라이선스로 공개 가능한 내용만 포함합니다.
- 링크 모음으로 대체하지 않고, 실제로 작동하는 스킬 문서를 작성합니다.
- 외부 전문자료는 원문 복사 대신 출처명, 링크, 확인일, 활용 범위를 기록합니다.
- 한국 사용자에게 익숙한 표현과 업무 흐름을 우선합니다.
- 특정 플랫폼의 내부 규칙보다 범용적인 에이전트 스킬 구조를 우선합니다.
- 스킬은 실제 업무에서 바로 호출할 수 있을 만큼 구체적으로 작성합니다.
- 법률, 세무, 의료 등 전문 영역에서는 AI의 한계와 책임 범위를 명확히 표시합니다.
